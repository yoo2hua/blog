<h2>Tokens First</h2>
<p>Design tokens are the atomic units of your system. Colors, spacing, typography, shadows — all named, all consistent. Start here before building any component.</p>
<pre><code class="language-ts">const tokens = {
  color: {
    foreground: '#ededed',
    background: '#0a0a0a',
    muted: '#a1a1aa',
  },
  space: {
    sm: '0.5rem',
    md: '1rem',
    lg: '2rem',
  },
};
</code></pre>
<h2>Component API Design</h2>
<p>A good component API is constrained. It offers flexibility without chaos. Document the intended use case. Say no to props that break the design intent.</p>
<h2>Versioning and Communication</h2>
<p>Design systems are products. They need versioning, changelogs, and migration guides. Communicate breaking changes early and often.</p>
<h2>Conclusion</h2>
<p>Scaling a design system is a social problem as much as a technical one. Invest in relationships, documentation, and governance. The code is the easy part.</p>
