<h2>Generic Constraints</h2>
<p>Use constraints to narrow generic types. This lets TypeScript infer more and catches misuse early.</p>
<pre><code class="language-ts">function pick&#x3C;T extends Record&#x3C;string, unknown>, K extends keyof T>(
  obj: T,
  keys: K[]
): Pick&#x3C;T, K> {
  const result = {} as Pick&#x3C;T, K>;
  for (const key of keys) {
    result[key] = obj[key];
  }
  return result;

<h2>Conditional Types</h2>
<p>Conditional types let you derive types from other types. They are the foundation of utilities like <code>Exclude</code>, <code>Extract</code>, and <code>ReturnType</code>.</p>
<pre><code class="language-ts">type IsString&#x3C;T> = T extends string ? true : false;
</code></pre>
<h2>Discriminated Unions</h2>
<p>Use discriminated unions for state machines. They make impossible states unrepresentable.</p>
<pre><code class="language-ts">type State =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'success'; data: string }
  | { status: 'error'; error: Error };
</code></pre>
